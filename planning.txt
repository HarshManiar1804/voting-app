voting app functionality

1. user sign in / sign up
2. see the list of candidates
3. vote one of the candidates, after voting, user should not be able to vote again.
4. there is a route which shows the list of candidates adn their live vote counts sorted by their vote count. 
5. user data must contain their one unique goverment id proof named: aadhar card number
6. there should be one admin who maintains the table of candidates and he can't be able to vote at any cost.
7. user can change their password
8. user can login using with aadhar card number and password only


--------------------------------------------------------------

Routes

User authentication :
    /signup : POST - create a new user account.
    /login : POST - log in to an existing account. [aadharcard + password]

Voting : 
    /candidates :  GET - get the list of candidates
    /vote/candidateId : POST - vote for specific candidate

Vote count :
    /vote/counts : GET - get the list of candidates sorted by their vote count.

User profile : 
    /profile : GET - get the user profile information.
    /profile/password : PUT - change the user password.

admin candidate management:
    /candidates : POST - create a new candidate
    /candidates/:candidateId : PUT - update an existing candidate
    /candidates/:candidateId : DELETE - delete an existing candidate from the list
